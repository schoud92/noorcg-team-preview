01 · Featured engagement

From spreadsheet sprawl to
one governed platform.

Sabra Health Care REIT — a public healthcare REIT — migrated their entire IT operation to Noor in March 2020, the week the pandemic hit. Seven-plus years later, we still run the platform their business operates on.

Sabra Health Care REIT · Public Healthcare REIT · Managed partner since 2020

One integrated Microsoft platform, run end to end.

The challenge

The March 2020 cutover was clean and flawless — and left the environment more secure. But the platform underneath the business was fragmented: critical finance and asset operations running across disconnected spreadsheets and tools, with no single governed system, no automation, and no shared source of truth.

What we built

We consolidated it onto Microsoft Cloud: roughly 20 custom SharePoint applications spanning Accounting, Investments, Lease Administration, Asset Management, and IT, plus an external operator self-service portal. Underneath run 240+ automated production workflows and Azure AI document extraction that reads lease abstracts, property assessments, and operator bank statements into structured data, with Power BI reporting on top — including a dedicated property-assessment platform managing hundreds of properties, with AI-assisted data entry and offline-capable field tools.

The outcome

A single, integrated platform run end to end — and a partnership we’ve operated as Sabra’s managed partner since 2020, now into its seventh-plus year.

02 · Inside the Sabra platform

Four workstreams from
one partnership.

The Sabra platform isn’t one project — it’s a stack of named systems we built and run. Here are four of them, pulled out on their own: an AI document pipeline, an approvals workflow, the finance dashboards leadership reads, and the API migration that kept it all online.

Lease Administration · Azure AI

Lease abstracts, read by AI into structured data.

Keying dense lease abstracts into the lease system by hand is slow, and every manual re-key is a chance to introduce an error into numbers that feed financial reporting. An Azure AI extraction pipeline now reads abstracts into structured fields, surfaced in a Lease Abstract Report — with a data-aware Claude assistant on top that answers questions against the live report and can pin specific rows by lease, so the numbers always trace back to source. The same approach now also reads property assessments and operator bank statements.

Azure AI · Claude assistant · Row-level pinningApplied AI ↗
Accounting · Workflow

Payment approvals, governed end to end.

Approving payments over email and spreadsheets leaves no clean audit trail — a real problem for a public company that has to show who approved what, and when. A Payment Approvals application on the same governed SharePoint platform now routes each request through its approval chain, with status and history captured in one place and a durable record behind every approval. No inbox archaeology.

Approval routing · Audit trailWorkflow automation ↗
Accounting · Power BI

AR aging and account balances, at a glance.

Receivables health and account balances were spread across systems, so the picture leadership needed had to be reassembled by hand each time it was asked for. AR-aging and account-balance reporting now sits on top of the consolidated platform — with care taken on which managed entities belong in each total, so the numbers reconcile rather than just look tidy. Finance reads from one consistent source instead of rebuilding the view every reporting cycle.

AR aging · Account balances · Reconciled totals
Platform / IT · Infrastructure

Re-homed the API the whole platform runs on.

Every application in Sabra’s operations depends on a shared back-end API — and moving it off an aging VM risked taking those apps offline if anything lost its path to the underlying data. We migrated it to an Azure App Service, diagnosed and restored the production connectivity an early cutover had missed, brought every dependent page back online on the new API, and retired the old host — with a tested rollback-and-restore playbook kept on the shelf.

Azure App Service · Zero-loss cutover · Rollback playbook
